## 1. 문서 목적
이 문서는 P0 범위를 실제 구현 티켓 단위로 분해한 작업 목록이다.

목표는 아래와 같다.
- 바로 착수 가능한 수준으로 작업을 쪼갠다.
- 선행 의존성과 병렬 가능 범위를 명확히 한다.
- 각 티켓의 완료 기준을 고정한다.

## 2. 운영 원칙
### 2.1 티켓 작성 원칙
- 한 티켓은 하나의 명확한 결과물을 가져야 한다.
- 프론트와 백엔드가 동시에 필요한 경우에도 사용자 가치 기준으로 묶는다.
- 티켓 설명만 읽고도 구현 시작이 가능해야 한다.

### 2.2 상태 기준
- `todo`: 아직 시작 전
- `in_progress`: 진행 중
- `blocked`: 선행 티켓 대기
- `done`: 수용 기준 충족

### 2.3 우선순위 기준
- Quick Draft까지의 경로를 가장 먼저 연다.
- 첫 화면 품질에 직접 영향 주는 티켓을 우선한다.
- 발행 후 하객 대응 기능은 Quick Draft와 공개 카드 뷰어 이후에 붙인다.
