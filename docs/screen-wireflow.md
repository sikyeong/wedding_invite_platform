# 화면별 와이어플로우

기준 날짜: `2026-03-08`

관련 문서:
- `docs/core-feature-spec.md`
- `docs/api-spec.md`

## 1. 문서 목적
이 문서는 모바일 청첩장 서비스의 P0 화면 흐름을 정의한다.

목표는 아래와 같다.
- 사용자가 어디서 진입하고 어디로 이동하는지 고정한다.
- 각 화면의 핵심 목적, 주요 액션, 필수 상태를 명확히 한다.
- 랜딩, Quick Draft, 에디터, 공개 카드가 같은 사용자 경험 축으로 이어지게 한다.
- 추천 preset 자동 적용과 sample preset 전환 흐름을 사용자 경험 안에 자연스럽게 녹인다.

## 2. 전체 흐름
제작자 흐름:
1. 랜딩 진입
2. Quick Draft 시작
3. 이름 입력 + 사진 업로드
4. 추천 preset 자동 적용 + draft 생성
5. draft 미리보기 확인
6. 필요하면 다른 sample preset으로 교체
7. 에디터 수정
8. 발행
9. 공개 카드 공유

하객 흐름:
1. 공유 링크 진입
2. 공개 카드 열람
3. 예식일시 / 장소 확인
4. RSVP 또는 방명록 작성
5. 계좌 복사 또는 길찾기

## 3. 공통 모바일 원칙
- 첫 화면에 핵심 정보와 첫 액션이 보여야 한다.
- 화면마다 CTA는 1개를 우선순위로 둔다.
- 복수 열 구조보다 세로 스택 흐름을 우선한다.
- 사용자는 스크롤을 통해 자연스럽게 내용을 이어서 봐야 한다.
- 미리보기와 실제 공개 카드의 시각적 차이는 최소화한다.
- sample preset 선택은 첫 미리보기 이후로 미루고, 초반 입력 폼은 가볍게 유지한다.

## 4. 제작자 화면 플로우
### 4.1 랜딩
#### 목적
- 서비스가 무엇인지 3초 안에 이해시킨다.
- Quick Draft 시작을 유도한다.

#### 진입
- 첫 방문
- 광고 / 검색 / 공유 링크

#### 필수 요소
- 서비스 한 줄 설명
- 사진 중심 가치 제안
- 준비된 sample preset 미리보기
- Quick Draft 시작 CTA
- 추천 preset 자동 적용 메시지
- 공개 후 가능한 기능 요약

#### 주요 액션
- `Quick Draft 시작`
- sample preset 미리보기 보기

#### 상태
- 기본 로딩 상태
- 콘텐츠 로드 완료 상태

#### 성공 조건
- 사용자가 별도 설명 없이 `사진 넣고 추천 preset에 합성된 초안을 본다`는 흐름을 이해한다.
- 첫 스크린 안에 CTA가 존재한다.

#### 연결 API / 이벤트
- 이벤트 `view_landing`
- 이벤트 `click_start_create`

### 4.2 Quick Draft 입력 화면
#### 목적
- 가장 짧은 입력 흐름으로 초안 생성에 필요한 정보만 받는다.

#### 진입
- 랜딩의 Quick Draft CTA 클릭

#### 필수 필드
- 신랑 이름
- 신부 이름
- 메인 사진 1장
- 갤러리 사진 4~40장

#### 선택 필드
- 중간 사진 1장

#### sample preset 처리 원칙
- 이 화면에서는 sample preset 선택을 강제하지 않는다.
- 보조 문구로 `샘플은 초안에서 바로 바꿀 수 있어요` 수준의 안내만 제공한다.

#### UI 구조
- 상단: 진행 설명 또는 짧은 보조 문구
- 상단 보조 문구: `추천 sample preset은 초안에서 자동 적용돼요`
- 본문: 이름 입력
- 본문: 메인 사진 업로드 슬롯
- 본문: 갤러리 업로드 슬롯
- 본문: 중간 사진 선택 업로드 슬롯
- 하단 고정 CTA: `초안 만들기`

#### 주요 액션
- 이름 입력
- 이미지 업로드
- 이미지 제거
- `초안 만들기`

#### 검증 상태
- 메인 사진 누락
- 갤러리 4장 미만
- 갤러리 41장 이상 시도
- 업로드 진행 중
- 생성 가능 상태

#### 오류 문구
- 갤러리 최대 초과 시
`갤러리 사진은 최대 40장까지 업로드할 수 있어요.`

#### 성공 조건
- 필수 입력만으로 CTA가 활성화된다.
- 생성 완료 후 곧바로 draft 미리보기로 이동한다.

#### 연결 API / 이벤트
- `POST /api/uploads/images`
- `POST /api/cards`
- 이벤트 `upload_photo_pack`
- 이벤트 `apply_recommended_preset`
- 이벤트 `auto_assign_photo_slots`

### 4.3 draft 생성 중간 상태
#### 목적
- 사용자가 시스템이 초안을 만들고 있음을 명확히 인지하게 한다.

#### 진입
- `초안 만들기` 클릭 직후

#### 필수 요소
- 진행 상태 메시지
- 업로드 / 카드 생성 진행 표시

#### 메시지 방향
- 복잡한 기술 설명보다 `추천 sample preset으로 첫 화면을 준비하고 있어요` 수준의 간결한 문구
- 대기 시간이 길지 않으면 스피너 + 짧은 문구만 사용

#### 성공 조건
- 생성 성공 시 자동으로 draft 미리보기로 이동한다.
- 실패 시 Quick Draft 입력 화면으로 복귀하고 입력값이 보존된다.

### 4.4 draft 미리보기 화면
#### 목적
- 사용자가 첫 화면과 전체 카드 무드를 바로 판단하게 한다.

#### 진입
- Quick Draft 생성 직후
- 소유자 전용 preview URL 접근

#### 필수 요소
- `main-parallax-screen`
- 갤러리
- 기본 생성 섹션
- 현재 적용된 sample preset 표시
- sample preset 전환 UI
- 상단 또는 하단의 편집 진입 CTA
- 발행 CTA

#### 주요 액션
- 카드 스크롤 보기
- 갤러리 확대 보기
- `다른 샘플 보기`
- `이 샘플 적용`
- `수정하기`
- `발행하기`

#### 상태
- draft 뷰
- 소유자 접근 가능 상태
- 토큰 만료 또는 접근 실패 상태

#### 성공 조건
- 첫 화면에서 사진과 이름이 자연스럽게 보인다.
- 사용자가 초안 품질을 즉시 판단할 수 있다.
- 사용자가 입력값 재작성 없이 다른 sample preset을 바로 바꿔볼 수 있다.
- 에디터와 발행으로 바로 이동할 수 있다.

#### 연결 API / 이벤트
- `GET /api/cards/:cardId/preview`
- `GET /api/presets`
- `PATCH /api/cards/:cardId/preset`
- 이벤트 `change_sample_preset`
- 이벤트 `view_main_parallax_screen`

### 4.5 에디터 홈
#### 목적
- 초안 이후 필요한 것만 빠르게 수정하게 한다.

#### 진입
- draft 미리보기의 `수정하기`

#### 필수 요소
- 카드 미리보기 영역 또는 바로가기
- 현재 적용된 sample preset 카드
- `샘플 바꾸기` 진입점
- 섹션 리스트
- 섹션 상태 표시: `기본값` / `수정됨` / `숨김`
- 섹션 순서 변경 핸들
- 발행 CTA

#### 주요 액션
- preset 확인
- sample preset 교체
- 섹션 열기
- 섹션 수정
- 섹션 숨김
- 섹션 순서 변경
- 미리보기 이동
- 발행 이동

#### 성공 조건
- 사용자가 현재 어떤 섹션을 수정해야 하는지 바로 파악한다.
- 수정 결과가 미리보기와 같은 데이터 구조로 반영된다.
- sample preset 변경 후에도 사용자 데이터는 유지되어야 한다.

#### 연결 API / 이벤트
- `GET /api/cards/:cardId/edit`
- `GET /api/presets`
- `PATCH /api/cards/:cardId/preset`
- `PATCH /api/cards/:cardId`
- `PATCH /api/cards/:cardId/gallery`
- `PATCH /api/cards/:cardId/sections`
- 이벤트 `open_editor`

### 4.6 섹션 편집 화면
#### 목적
- 각 섹션을 독립적으로 수정하게 한다.

#### 대상 섹션
- 예식일시
- 예식장소
- D-Day
- 부모님 성함
- 계좌
- 인터뷰/스토리
- RSVP 노출
- 방명록 노출

#### 공통 UI 구조
- 상단: 뒤로가기, 섹션명
- 본문: 섹션별 입력 필드
- 하단: 저장 CTA
- 보조 액션: 숨김 처리

#### 공통 상태
- 기본값
- 수정 중
- 저장 완료
- 숨김
- 검증 오류

#### preset 전환 규칙
- `edited` 상태 섹션은 sample preset 전환 시 유지된다.
- `default` 상태 섹션은 새 preset의 기본값으로 바뀔 수 있다.

#### 성공 조건
- 저장 후 즉시 에디터 홈과 미리보기에 반영된다.
- 숨김 처리 후 카드 렌더에서도 제외된다.

### 4.7 발행 확인 화면
#### 목적
- 공개 전 마지막 확인과 공유 URL 생성을 수행한다.

#### 진입
- draft 미리보기의 `발행하기`
- 에디터의 `발행하기`

#### 필수 요소
- 공개 URL slug 입력 또는 확인
- 현재 적용된 sample preset 요약
- 대표 공유 정보 요약
- 발행 CTA

#### 주요 액션
- slug 수정
- `발행 완료`

#### 검증 상태
- slug 중복
- 발행 가능 상태
- 발행 실패 상태

#### 성공 조건
- 발행 성공 즉시 공개 URL을 확인할 수 있다.
- 공유 액션으로 바로 이어질 수 있다.

#### 연결 API / 이벤트
- `POST /api/cards/:cardId/publish`
- 이벤트 `publish_card`

### 4.8 발행 완료 화면
#### 목적
- 제작자에게 공개가 완료되었음을 명확히 보여준다.

#### 필수 요소
- 공개 URL
- 복사 버튼
- 카카오 공유 버튼
- 카드 바로보기 버튼
- 적용된 sample preset 정보 요약

#### 성공 조건
- 사용자가 추가 설명 없이 바로 공유할 수 있다.

## 5. 하객 화면 플로우
### 5.1 공개 카드 진입 화면
#### 목적
- 하객이 카드의 첫인상을 받는 핵심 화면이다.

#### 진입
- 카카오 공유
- 링크 직접 접근
- SNS / 메시지 앱 공유

#### 필수 요소
- `main-parallax-screen`
- 신랑/신부 이름
- 스크롤 유도감

#### 성공 조건
- 첫 화면에서 카드 품질과 예식 초대 의도가 명확히 전달된다.
- 적용된 sample preset의 톤이 공개 카드에서도 일관되게 유지된다.

#### 연결 API / 이벤트
- `GET /api/public/cards/:slug`
- 이벤트 `view_main_parallax_screen`

### 5.3 예식 정보 확인 섹션

## 6. 화면 상태 표준
### 6.1 로딩 상태
- 본문 레이아웃이 크게 흔들리지 않아야 한다.
- 메인 이미지는 우선 로딩 대상으로 본다.

### 6.2 빈 상태
- Quick Draft 입력 전에는 업로드 슬롯이 분명히 보여야 한다.
- 방명록이 비어 있으면 첫 메시지 작성을 유도한다.

### 6.3 오류 상태
- 사용자가 무엇을 수정해야 하는지 바로 이해할 수 있어야 한다.
- 네트워크 오류는 재시도 동선을 제공한다.

### 6.4 성공 피드백
- RSVP 제출, 방명록 작성, 계좌 복사, preset 변경, 발행 완료는 즉시 피드백을 보여야 한다.

## 7. 화면 간 전환 규칙
- 랜딩 -> Quick Draft: 직접 전환
- Quick Draft -> 생성 중간 상태: CTA 클릭 직후
- 생성 중간 상태 -> draft 미리보기: 생성 성공 시 자동 전환
- draft 미리보기 -> 다른 sample preset: 같은 화면 안에서 즉시 전환
- draft 미리보기 -> 에디터: `수정하기`
- draft 미리보기 -> 발행 확인: `발행하기`
- 에디터 -> draft 미리보기: 미리보기 CTA
- 에디터 -> 다른 sample preset: preset 교체 액션
- 에디터 -> 발행 확인: 발행 CTA
- 발행 완료 -> 공개 카드: 바로보기 CTA

## 8. 수용 기준
### 8.1 제작자 흐름
- 랜딩 진입 후 Quick Draft 시작까지 혼란이 없어야 한다.
- 필수 입력만으로 draft 미리보기에 도달할 수 있어야 한다.
- 추천 preset이 자동 적용되어야 한다.
- sample preset 교체가 입력값 재작성 없이 가능해야 한다.
- 에디터에서 섹션 수정, 숨김, 순서 변경이 가능해야 한다.
- 발행 후 즉시 공개 URL을 얻을 수 있어야 한다.

### 8.2 하객 흐름
- 공개 카드 첫 화면에서 사진과 이름이 자연스럽게 보여야 한다.
- 적용된 sample preset의 톤과 첫인상이 깨지지 않아야 한다.
- 일정, 장소, RSVP, 방명록, 계좌 복사 핵심 행동이 모바일에서 무리 없이 수행되어야 한다.
- 갤러리 확대 모달은 닫기와 이동이 직관적이어야 한다.
