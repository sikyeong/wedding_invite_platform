# AGENTS.md

## 0. 문서 목적
이 문서는 `한국형 모바일 청첩장 서비스`를 새로 구현할 때 흔들리지 않기 위한 `핵심 실행 브리프`다.
아이디어 수집 문서가 아니라, 제품 방향과 구현 우선순위를 짧고 명확하게 고정하는 용도다.

기준 날짜: `2026-03-08`

## 1. 제품 한 줄 정의
`웨딩화보를 가장 깔끔하고 세련되게 보여주고, 이름 + 사진 세트만으로 90초 안에 추천 preset 기반 모바일 청첩장 초안을 확인하게 만드는 서비스`

## 2. 이 제품의 본질
이 서비스의 핵심은 기능 나열이 아니다.
고객이 `신랑/신부 이름 + 웨딩 사진`을 넣는 즉시,
`우리 커플에게 어울리는 preset에 합성된 모바일 청첩장 첫인상`을 빠르게 보게 만드는 것이 핵심이다.

즉, 이 제품은 다음 순서로 평가받는다.
1. 첫 화면이 예쁜가
2. 사진이 살아나는가
3. 복잡하지 않고 바로 써볼 수 있는가
4. 발행 후 하객 대응 기능이 충분한가

## 3. 고객이 첫 방문에서 느껴야 하는 인상
첫 방문 인상은 반드시 아래 3가지를 전달해야 한다.
- `깔끔함`
- `세련됨`
- `편의성`

의미:
- 깔끔함: 정보가 적절히 정리되어 있고, 화면이 복잡하지 않아야 한다.
- 세련됨: 사진, 타이포, 여백, 색이 촌스럽지 않고 브랜드처럼 보여야 한다.
- 편의성: 무엇을 먼저 해야 하는지 명확하고, 초안 확인까지의 단계가 짧아야 한다.

레퍼런스 사이트 `https://theirmood.com/` 는 디자인 톤, 글씨체 등의 `무드 참고`만 한다.
특정 서비스의 카피를 그대로 복제하지 않는다.

## 4. P0 핵심 기능: Quick Draft / sample preset / 갤러리 기능 / 여러 샘플 카드

### Quick Draft
#### 목적
고객이 `사진 세트 + 이름`만으로 준비된 sample preset에 자기 데이터를 갈아끼운 결과를 바로 판단하게 한다.

#### 입력 규칙
필수 입력:
- 신랑 이름
- 신부 이름
- 메인 사진 1장
- 갤러리 사진 4장 이상, 최대 40장

선택 입력:
- 중간 사진 1장

중간 사진 기본 동작:
- 별도 업로드가 없으면 갤러리 사진 1장을 자동 할당한다.

#### sample preset 정책
- 초안 보기 전에 sample preset 선택을 강제하지 않는다.
- Quick Draft 생성 시 시스템이 `추천 preset 1개`를 자동 적용한다.
- 추천은 완전 랜덤만으로 끝내지 않는다.
- 첫 미리보기 직후 사용자는 `3~5개` 정도의 준비된 sample preset 중 다른 것을 즉시 바꿔볼 수 있어야 한다.
- sample preset 전환 시 `이름 / 사진 / 하객 응대 데이터`는 유지되고, 레이아웃과 표현 계층만 교체되어야 한다.
- sample preset은 아래의 기본값을 제공할 수 있다.
  - 레이아웃
  - 타이포
  - 색감
  - 소개문구
  - 섹션 기본 노출 규칙
  - 카드 애니메이션
  - 배경 음악

#### 결과
Quick Draft 완료 직후 아래가 즉시 보여야 한다.
- `추천 preset` 적용
- `main-parallax-screen` 생성
- 기본 갤러리 자동 구성
- 중간 섹션 자동 구성
- 비공개 미리보기 가능
- sample preset 교체 가능

#### 시간 목표
- 첫 입력부터 첫 미리보기까지 `90초 이내`

#### 금지사항
- 초안 보기 전에 날짜/시간/장소/계좌/공지 같은 부가 입력을 강제하지 않는다.
- 초안 보기 전에 복잡한 옵션 설정을 요구하지 않는다.
- 초안 보기 전에 sample preset 비교 탐색을 길게 강제하지 않는다.

#### 갤러리 제한 정책
- 갤러리 최대 업로드 수는 `40장`
- 41번째 업로드는 즉시 차단
- 안내 문구는 아래 문구를 그대로 사용
  - `갤러리 사진은 최대 40장까지 업로드할 수 있어요.`

### 갤러리 기능 / 여러 샘플 카드 참고 자료.
`ref/` 경로의 파일내용을 참고한다. (기능은 그대로 써도 된다.)

## 5. 에디터 원칙
에디터는 `모든 정보를 한 번에 입력하는 화면`이 아니라,
`초안을 먼저 보고 필요한 것만 다듬는 화면`이어야 한다.
또한 `preset 기반으로 생성된 초안`을 빠르게 수정하는 화면이어야 한다.

원칙:
- 초안 확인 후 세부 정보 입력
- 미리보기와 실제 렌더가 최대한 동일해야 함
- 섹션은 독립 모듈이어야 함
- 섹션 순서 변경 가능해야 함
- preset 교체 가능해야 함
- 사용자 데이터와 preset 표현은 분리되어야 함
- 모바일에서 먼저 편하게 써야 함

Quick Draft 직후 자동 생성되는 기본 섹션:
- 예식일시
- 예식장소
- D-Day
- RSVP
- 방명록
- 부모님 성함
- 계좌 섹션
- 인터뷰/스토리

각 섹션 상태:
- `기본값`
- `수정됨`
- `숨김`

예외:
- `main-parallax-screen` 와 `gallery` 는 시스템 필수 섹션이라 P0에서 숨김/삭제할 수 없다.
- `RSVP` 는 Quick Draft 시 기본 생성되고 기본 노출되지만, 필요하면 에디터에서 숨길 수 있다.

## 7. 랜딩 페이지 원칙
랜딩은 회사 소개 페이지가 아니라 `즉시 써보고 싶게 만드는 제품 시작 화면`이어야 한다.

랜딩의 핵심 역할:
- 서비스가 무엇인지 3초 안에 이해시킨다.
- `사진 넣으면 준비된 sample preset에 바로 합성된 결과를 볼 수 있다`는 점을 분명히 보여준다.
- 첫 CTA는 Quick Draft 시작이어야 한다.

랜딩에서 강조할 메시지:
- 사진 중심 첫인상
- 빠른 초안 생성
- 추천 preset 자동 적용
- 첫 미리보기 이후 sample preset 교체 가능
- 깔끔하고 세련된 결과물
- 발행 후 RSVP/방명록/공유까지 가능

랜딩에서 약해야 하는 것:
- 장황한 기능 카탈로그
- 복잡한 가격 선택지 (가격은 무료, 혹은 6900원의 워터마크 제거 및 결혼식 이후 100일까지 사용가능 권)
- 초반부터 과한 입력 폼
- 초반부터 과한 sample preset 선택 UI

## 8. 디자인 원칙
디자인은 반드시 `모바일 퍼스트`로 본다.

지켜야 할 방향:
- 여백이 충분해야 한다.
- 타이포는 단정하고 읽기 쉬워야 한다.
- 색은 절제되어야 한다.
- 카드, 버튼, 배지 남발 금지
- “템플릿 쇼핑몰”처럼 보이면 안 된다.
- “브랜드 서비스”처럼 보여야 한다.

톤 키워드:
- clean
- refined
- calm
- photo-first

피해야 할 것:
- 싸보이는 파스텔 남발
- 복잡한 박스 나열
- 무거운 기능표
- 기본 웹앱 같은 투박한 입력 UI

## 9. 기술 구현 원칙
프론트엔드:
- 랜딩 / 에디터 / 카드 뷰어를 분리한다.
- 섹션은 재정렬 가능한 모듈로 만든다.
- 미리보기와 실제 렌더를 가능한 한 같은 데이터 구조로 만든다.
- card 데이터와 preset 표현 계층을 분리한다.
- preset 전환 시 사용자 데이터는 유지되고 표현만 교체되게 만든다.

백엔드:
- 카드 조회 API와 편집 API 분리
- 갤러리 40장 제한은 서버에서도 강제
- draft와 published 상태를 명확히 구분
- draft 카드도 소유자 token이 있으면 비공개 미리보기 허용 가능
- `preset_id`, `preset_version`, 사용자 override를 분리 저장한다.
- 렌더 응답은 preset 기본값과 사용자 override를 합성한 결과를 내려줘도 된다.

성능:
- 첫 화면 필수 정보 우선 렌더
- 메인 이미지 로딩 안정성 우선
- 카카오 인앱 브라우저 호환 필수

## 11. 성공 기준
이 제품의 성공 기준은 아래다.
- 고객이 빠르게 초안을 본다.
- 첫 화면이 예쁘다고 느낀다.
- 발행까지 이탈하지 않는다.
- 하객이 RSVP / 길찾기 / 계좌복사 같은 핵심 행동을 실제로 한다.

가장 중요한 KPI:
- `TTFD(Time To First Draft)`
- `Quick Draft 성공률`
- `추천 preset 유지율`
- `초안 -> 발행 전환율`
- `view_main_parallax_screen` 이후 이탈률

## 12. 새로 시작할 때의 우선순위
새 구현은 아래 순서로 진행한다.

1. 랜딩 최소 버전

- Quick Draft CTA
- sample preset 최소 노출

2. Quick Draft 완성
- 이름 입력
- 메인 사진 업로드
- 갤러리 4~40장 업로드
- 추천 preset 자동 적용
- 첫 미리보기에서 sample preset 교체
- 중간 사진 자동/선택 처리
- 즉시 미리보기

3. 카드 뷰어 완성
- `main-parallax-screen`
- 갤러리
- 날짜/장소

4. 에디터 완성
- 기본 섹션 수정
- 섹션 ON/OFF

- 공유 설정

5. 발행/운영
- publish
- draft preview
- 통계
